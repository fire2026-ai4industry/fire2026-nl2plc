Evaluation

Participant systems will be evaluated on the official test dataset using automated metrics designed to measure both syntactic correctness and semantic fidelity of generated Structured Text programs.

Task A: Natural Language to Structured Text

Task A evaluates the ability of a system to generate IEC 61131-3 Structured Text (ST) programs from natural language descriptions.

Metrics

Metric Description
Syntax Correctness (SC) Parser-based validation of generated ST code.
Structural Match Score (SMS) Comparison of conditions, timers, assignments and control structures with reference code.
Token Similarity (TS) Measured using BLEU / CodeBLEU style metrics.
Task A Score = 0.3 × SC + 0.5 × SMS + 0.2 × TS

Task B: PLC/ST Code Correction & Refinement

Task B evaluates the ability of a system to identify and correct syntactic and structural issues in Structured Text programs using the original natural language description as guidance.

Metrics

Metric Description
Syntax Correctness (SC) Parser-based validation of corrected ST code.
Error Reduction Score (ERS) Normalized reduction in syntactic and logical errors.
Structural Alignment (SA) Alignment between corrected code and reference implementation.
Task B Score = 0.3 × SC + 0.3 × ERS + 0.4 × SA

Final Ranking

The final ranking of participating teams will be determined by combining scores from both subtasks.

Final Score = 0.7 × Task A Score + 0.3 × Task B Score

Tie-Breaking Policy

In the event of a tie, the team with the higher Task A score will be ranked higher. Additional tie-breaking criteria may be announced if required.

Evaluation Toolkit

The official evaluation toolkit and scoring scripts will be released before the test phase.
