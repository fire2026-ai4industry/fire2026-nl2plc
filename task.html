Task Description

Motivation

Programmable Logic Controllers (PLCs) are widely used in industrial automation systems such as manufacturing, energy, transportation, and process industries. PLC programming requires specialized expertise and knowledge of industrial control systems.

Domain experts and plant operators often describe control logic in natural language, creating a significant gap between system requirements and actual implementation. Errors in PLC programs may lead to safety hazards, production losses, and equipment damage.

This shared task aims to bridge this gap by encouraging the development of AI-based systems capable of generating and refining IEC 61131-3 Structured Text (ST) programs from natural language descriptions.

Use Case Example

Natural Language Description

If temperature exceeds 80°C and pressure is above threshold, stop the motor and trigger alarm after 5 seconds delay.

Equivalent Structured Text Program 
IF (Temperature > 80) AND
   (Pressure > Pressure_Threshold) THEN

   Motor := FALSE;

   AlarmTimer(
       IN := TRUE,
       PT := T#5s
   );

ELSE

   AlarmTimer(IN := FALSE);

END_IF;

IF AlarmTimer.Q THEN
   Alarm := TRUE;
END_IF;

Shared Task Structure

The shared task consists of two complementary subtasks.

Task A: Natural Language to Structured Text

Generate IEC 61131-3 compliant Structured Text programs from natural language descriptions of industrial control logic.

Input

  • Natural Language Description of industrial control logic

Output

  • IEC 61131-3 Structured Text (ST) Program

Objective

  • Generate syntactically correct ST code
  • Preserve intended control logic semantics
  • Produce executable industrial automation programs

Task B: PLC/ST Code Correction & Refinement

Detect and correct syntactical errors in Structured Text code using the corresponding Natural Language Description as the reference standard.

Input

  • Generated Structured Text code
  • Original Natural Language Description

Output

  • Corrected IEC 61131-3 compliant Structured Text program

Objective

  • Identify syntax errors
  • Correct malformed ST programs
  • Improve semantic alignment with requirements

Allowed Resources

  • Open-source language models
  • Publicly available datasets
  • Open-source software libraries
  • Custom training and fine-tuning approaches

LLM Usage Policy

Only open-source models up to 8 GB in size are permitted for participation.
